The KFSST package

KFSST is an add-on package for the statistical environment R to efficiently estimate movement parameters and predict the most probable track from raw light-based geolocations and sea surface temperatures (SST). This package offers an approach to use both types of information in one coherent state-space model and carries out the estimation via the extended Kalman filter (KF). Read all about the model here (html / pdf).

How to try the package

The KFSST package depends on a few other packages (date, akima, locfit, and kftrack).

To make installation a bit easier, a script has been prepared to set up all of these five packages.

To run it fire up R and type (or paste) the following line: 
  source('ftp://ftp.soest.hawaii.edu/PFRP/tracking/install')
This should be done in root/administrator-mode on Linux/Windows to allow each package to add to the R-documentation and library tree.

How to use

KFSST comes with a function explaining how to fit a track. It covers all steps from how to read in the raw track, automatically obtain SST-field, smooth the field, fit the track, and plot and investigate the final track. This help is displayed by typing: 
  road.map()
Also the package comes with a complete example. To see it type: 
  example(blue.shark)

Hints: 
1) Try using the 'plotmap' function with the argument  	 
   'zoom=TRUE', which makes it possible to zoom in on  	 
   a sub-map by left-clicking twice on the map to      	 
   select the area. Once the area is selected simply   	 
   right-click. 				       	 
						       	 
2) Try using 'write.sst.field' function with the       	 
   argument 'peak=TRUE' to visually inspect the        	 
   each of the observed SST-fields and their 	       	 
   corresponding smoothed SST-field. This is helpful   	 
   when trying to decide the degree of smoothing to    	 
   use. 					       	 
						       	 
3) Sometimes the raw latitude measurements are very    	 
   poor and completely useless. One way to ignore the  	 
   latitude information is to assign a huge fixed      	 
   variance to these measurements. For instance like:	 
  							 
     fit<-kfsst(..., var.struct='uniform', 	       	 
                sy.active=FALSE, sy.init=1000) 		 
							 
4) SST-data matching your track can be automatically 	 
   obtained by typing: 					 
  							 
     path<-get.sst.from.server(track)			 
							 
5) To get one SST-field covering a set of tracks use 	 
   a command similar to:       				 
							 
     path<-get.sst.from.server(list(track1, track2, ...))

6) The land-polygons can be saved for other purposes 
   from the function 'plotmap' via the option 
   'save=TRUE'.
